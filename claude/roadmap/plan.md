# Claude 学习路线

> 目标：从基础认知到可复用工作流，再到小型实战。
> 背景：10 年开发经验，日常用 AI 辅助编码和写文档，未系统学习过提示词工程。

---

## 阶段 1：基础认知（5 天，每天 40 分钟）

### 第 1 天：建立提示词结构意识
- 核心：Claude 的决策机制、黄金结构（角色 + 任务 + 约束 + 格式）、上下文窗口
- 练习：用同一个任务写 3 个版本的提示词，对比输出差异
- 产出：`lessons/claude/01-overview.md`

### 第 2 天：掌握约束与输出控制
- 核心：负向约束、格式控制、长度/粒度控制、Few-shot 示例
- 练习：让 Claude 把同一段技术文档改写成 2 种风格，对比效果
- 产出：`lessons/claude/02-constraints.md`

### 第 3 天：编码场景深度实践
- 核心：代码生成/修改/解释三种模式、上下文注入、迭代对话、Claude Code vs 网页版
- 练习：让 Claude 定位 bug + 生成文档 + 重构函数
- 产出：`lessons/claude/03-coding.md`

### 第 4 天：复杂任务拆解与链式提示
- 核心：任务分解、链式提示、自我验证、常见陷阱
- 练习：完成一个 4 步端到端任务（分析 → 大纲 → 章节 → 自查）
- 产出：`lessons/claude/04-chaining.md`

### 第 5 天：构建个人提示词模板库
- 核心：提示词复用与参数化、场景分类、模板迭代、团队协作
- 练习：提炼 5-8 个高频模板，每个包含场景、模板正文、变量说明、使用示例、已知限制
- 产出：`prompts/` 目录下的模板文件

### 阶段 1 产出清单

- [ ] 1 份作业（hw-01）
- [ ] 1 套 prompts 模板库

## 阶段 2：工具与工作流（2-4 天）
- 与代码仓库协作：读写文件、分步骤产出
- 结合日常流程：roadmap → lessons → practice → homework → review
- 任务拆解与验收标准

产出：
- 1 个小型流程演练项目
- 1 份复盘记录（review/）

## 阶段 3：小实战（3-5 天）
- 选一个真实需求：学习某技术/整理资料/做小项目
- 从 0 到 1 走完整流程

产出：
- 完整的 lessons + practice + homework + review
