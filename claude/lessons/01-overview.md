# Claude 概览

## 关键概念
- Claude 是对话式 AI，擅长语言理解、写作、总结与推理
- 输出质量取决于输入质量：目标清晰、约束明确、格式指定
- 最有效的提示词结构：**角色 + 任务 + 约束 + 示例**

---

## 提示词黄金结构详解

| 要素 | 作用 | 缺失时的后果 |
|---|---|---|
| **角色** | 限定 AI 的专业视角和表达方式 | 输出泛泛，缺乏专业深度 |
| **任务** | 明确要完成什么 | AI 猜测意图，输出跑偏 |
| **约束** | 边界条件：时间、长度、风格、禁忌 | 输出不可用，需要大量返工 |
| **格式** | 指定输出结构（表格/清单/代码块） | 格式混乱，需要重新整理 |
| **示例**（可选） | 用 1 个例子校准风格和质量 | AI 按自己理解发挥，结果不稳定 |

---

## 最小示例

需求：让 AI 写一段学习计划。

提示词：
```
你是学习教练。请为"学习 Git 基础"制定 7 天学习计划，要求：
1. 每天 30 分钟
2. 输出为表格
3. 每天包含目标与练习
```

## 错误示范 vs 正确示范

| 场景 | 错误示范 | 正确示范 | 差异原因 |
|---|---|---|---|
| 学一个概念 | "帮我学 Claude" | "你是 AI 工具导师。请为我（10 年开发经验）制定 5 天 Claude 学习计划，每天 40 分钟，输出为 Markdown 表格，含目标/练习/验收标准，最后一天产出提示词模板库" | 补充了角色、背景、时间、格式、最终产出 |
| Review 代码 | "帮我看看这段代码" | "你是资深 Python 工程师。请 review 以下函数，重点关注：① 边界条件 ② 性能瓶颈 ③ 可读性。用表格输出，每行一个发现，含问题/严重等级/建议" | 指定了视角、关注维度、输出格式 |
| 写技术文档 | "帮我写个文档" | "你是技术文档工程师。请为以下函数写 API 文档，格式：函数名、参数表、返回值、异常、示例代码。目标读者是 2 年经验的初级工程师" | 指定了格式结构、目标读者 |

## 常见误区
- **问题太泛**：比如"教我编程" — AI 不知道从哪开始
- **没有约束**：不说明时间、输出格式、难度 — 输出不可用
- **缺少上下文**：不说你的背景或目标 — 内容太浅或太深
- **一次问太多**：把 5 个问题塞进一个 prompt — 后面的问题被忽略
- **不指定格式**：让 AI 自由发挥 — 需要大量手动整理

## 练习题

**练习 1：** 把下面问题改写成更高质量的提示词：
> "帮我学 Claude。"

参考输出：
```
你是一位 AI 工具导师，擅长用实战驱动的方式教学。

请帮我制定一份"从零到熟练使用 Claude"的学习计划，要求：
1. 周期 5 天，每天 40 分钟
2. 输出为 Markdown 表格，包含：天数、目标、核心知识点、动手练习、验收标准
3. 我的背景：有 10 年开发经验，日常用 AI 辅助编码和写文档，但对提示词工程没有系统学习过
4. 每天至少包含一个"错误示范 vs 正确示范"的对比
5. 最后一天产出一份我自用的提示词模板库
```

**练习 2：** 把下面问题改写成更高质量的提示词：
> "帮我看看这段代码有没有问题。"

（提示：加上角色、关注维度、输出格式、你的上下文）

如何提问
```
对照黄金结构（角色 + 任务 + 约束 + 格式），我有没有漏掉或不够具体的地方？
如果要再上一个台阶，你会建议我加什么？
```

参考输出：

```
你是资深前端架构师，请Review这段代码，重点关注：边界条件、性能瓶颈、可读性。用表格输出，每行一个发现，包含问题、严重等级、建议、具体位置
```

优化后的输出：
```
你是资深前端架构师，有 10 年 React 项目经验。
请 Review 以下 React 组件代码，重点关注：
① 边界条件（空值、异常输入）
② 性能瓶颈（不必要的重渲染、大列表处理）
③ 可读性（命名、函数拆分）

要求：
1. 不要改动业务逻辑，只关注工程质量
2. 最多提 5 个问题，按严重等级排序
3. 用表格输出，每行一个发现，包含：问题位置 / 问题描述 / 严重等级（高/中/低）/ 修复建议
4. 最后用一句话总结整体代码质量

我的背景：10 年前端经验，这个组件将用于生产环境。
```

```
你是 Git 专家。请告诉我如何把最近 3 次 commit 合并成一个，要求：

我的背景：熟悉 Git 日常操作，只是这个命令记不清了。

```
